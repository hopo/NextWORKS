- 다형성 : 부모 클래스에서 물려받은 가상 함수를 자식 클래스 내에서 오버라이딩 되어 사용되는 것


# 재귀호출에 대한 설명 / 팩토리얼 코드
- 재귀호출 : 하나의 함수에서 자신을 다시 호출하여 작업을 수행하는 방식. 되부름 이라고 불리기도 한다.
- 팩토리얼 코드 :
```
public class Factorial {
	public static void main(String[] args) {
		int result = fact(input);
		System.out.println(result);
	}

	public static int fact(int n) {
		if (n <= 1) {
			return n;
		} else { 
			return fact(n-1) * n;
		}
	}
}
```

# 1~n 합 loop / 가우스 / 차이점
- loop & gaus :
```
public class TotalSum {
	public static void main(String[] args) {
		int n = 10;

		int ex1 = loop(n);
		System.out.printf("ex1: %d\n", ex1);
		int ex2 = gaus(n);
		System.out.printf("ex2: %d\n", ex2);

	}
	static int loop(int n) {
		int i = 1;
		int ret = 0;

		while (i <= n) {
			ret += i;
			i++;
		}
		return ret;
	}
	static int gaus(int n) {
		int ret = (1 + n) * (n / 2);
		if (n % 2 != 0) {
			ret += n / 2 + 1;
		}
		return ret;
	}
}

```

- 차이점 :
 내부에서 돌아갈 때 루프문은 GOTO가 반복적으로 실행되고, 가우스는 단순 산술연산으로 끝난다.

# 싱글톤패턴 / 언제 사용하는가
- 싱글톤패턴 : 클래스 외부에서 여러개의 객체 생성을 못하게 클래스 내부에서 인스턴스를 제공하는 거. 인스턴스를 하나만 만들 수 있게 하는 패턴.
- 언제 사용하는가 :
 고정된 메모리 영역을 얻으면서 한번의 객체 생성으로 인스턴스를 사용하여 메모리 낭비를 줄일 때

# 이너 아우터 유니온 유니온올
- 이너조인 : A와 B의 교집합. 벤다이어그램으로 그렸을 때 교차되는 부분.
- 아우터조인 : A와 B의 합집합. 벤다이어그램으로 그렸을 때 합집합 부분.
- 유니온 : 집합 A와 집합 B을 상하로 결합하여 하나의 집합으로 만드는 것.
- 유니온올 : 유니온에서 중복을 제거 하지 않은 것.

# mes / hmi
- Manufacturing Execution System : 생산 현장에서 작업을 수행하기 위한 활동을 지원하기 위한 시스템으로 생산현장의 데이터 수집 및 정보의 전달등을 통하여 관리자와 작업자의 의사결정 및 지시 보고 등을 지원하는 것
- Human Machine Interface : 운영자에게 설비의 처리 데이터를 표시해 주고 운영자가 처리 과정을 제어할 수 있도록 하기 위한 장치.

# 스택 / 힙 / 데이터
- 스택 : 함수 호출 시 생성되는 지역 변수와 매개 변수가 저장되는 영역
- 힙 : 필요에 의해 동적으로 메모리를 할당 할 때 사용되는 저장 역역
- 데이터 : 프로그램의 시작과 동시에 할당되고, 프로그램이 종료되어야 메모리에서 소멸됨

# 전역 / 지역 / 상수
- 전역 : 데이터 영역에 적재되고 어디에서든 접근 가능
- 지역 : 스택 영역에 적재되고 지정된 스코프에서만 접근 가능
- 상수 : 변하지 않는 값

# 10진수 -> 16진수/8진수
- 10진수 18을 16진수로 : 12
- 16진수 18을 10진수로 : 24
- 8진수 42을 10진수로 : 34

# 데이터웨어하우스, 데이터마트, 데이터 마이닝
- 데이터웨어하우스 : 사용자의 의사 결정에 도움을 주기 위하여, 기간시스템의 데이터베이스에 축적된 데이터를 공통의 형식으로 변환해서 관리하는 데이터베이스
- 데이터마트 : 데이터웨어하우스에서 데이터를 꺼내 사용자에게 제공하는 역할
